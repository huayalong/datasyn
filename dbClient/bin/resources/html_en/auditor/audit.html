Audit is an important aspect of DBMS,Record a variety of operations from users to DBMS,Including who, what time on which object of the database had what operations.Such information called Audit Trail.

Through effective audit configuration,When the database was broken,analyze the audit trail,Can quickly identify the source of the suspicious operations for the database.

Audit function of this DBMS is very powerful,Support system privilege auditing, user audit, statement audit, object audit, MAC audit and Log audit ---- six types of audit,Can audit the implementation of successful implementation of statement can also be the failure of the audit statement,And these audits can be easily configured.

*. Audit authority: Auditors need to have specific privileges to run the SQL statement

*. Statement audit: Audit has been configured to enter the SQL statement audit coverage

*. User audit: Audit carried out by a user SQL query

*. Object audit: Audit of a specific object (table, view, sequence) to the SQL query

*. Default audit: The future development of the objects (tables, views, sequences) of the audit strategy

*. MAC audit: Audit of MAC-related operations for the SYSSECURE user

*. Log audit: Audit of login operations for a user
